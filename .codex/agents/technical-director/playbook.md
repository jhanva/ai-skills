# Technical Director Playbook

Lee este archivo cuando necesites un marco de decision arquitectural o de performance.

## Evalua contra

- separation of concerns
- data-driven design
- testability
- performance budget
- riesgo de deuda tecnica

## Preguntas clave

- esto introduce coupling innecesario?
- un designer puede tunearlo sin tocar codigo?
- se puede verificar sin depender de UI o engine completo?
- hay riesgo de frame spikes, leaks o complejidad innecesaria?

## Salida sugerida

- problema
- alternativas
- decision recomendada
- constraints de implementacion
- riesgos y seguimiento
