# QA Analyst Playbook

Lee este archivo cuando necesites clasificar bugs, armar test plans o cerrar un smoke test.

## Severidades

- `S1`: crash, data loss, blocker
- `S2`: feature rota o regression fuerte
- `S3`: glitch menor o comportamiento degradado
- `S4`: polish, typo, detalle visual

## Divide el trabajo

- automatizado: logica deterministica, parsing, state transitions, save/load
- manual: game feel, timing, audio, UI, balance, claridad

## Modos comunes

- test plan
- bug report
- smoke test
- regression check
- playtest protocol

## Entregable sugerido

- scope
- pasos
- expected vs actual
- severidad
- evidencia o riesgo
- conclusion PASS/FAIL si aplica
