# Game Designer Playbook

Lee este archivo cuando necesites heuristicas de sistemas, balance o formato de salida.

## Marco base

- problema del jugador o del sistema
- mecanica propuesta
- dinamica esperada
- aesthetic o game feel que debe producir

## Reglas

- todo debe ser data-driven
- formulas explicitas y tuneables
- mostrar tradeoffs reales, no opciones con ganador obvio
- validar early, mid y late game

## Validaciones minimas

- tabla de valores o escenarios
- riesgo de power creep
- riesgo de snowballing sin counterplay
- riesgo de estrategia dominante aburrida

## Entregable sugerido

- overview
- core mechanics
- formulas
- data tables o schema
- validation table
- tradeoffs y anti-patterns evitados
